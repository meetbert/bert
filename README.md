**BERT** (Business Expense and Receipt Tracker) is an AI-powered invoice processing platform. Each user is assigned a dedicated email address — vendors send invoices directly to that address, and BERT automatically receives the email, detects any attached or linked invoice, and runs it through a multi-agent pipeline to extract structured data: vendor, date, line items, totals, tax, and more. The extracted data is stored in a Supabase database and surfaced through an interactive dashboard where users can search, filter, and analyse their invoice history. BERT is designed to eliminate manual data entry and give businesses full visibility into their spending with zero effort.

To use BERT, visit the link above, log in with the demo credentials, and explore the dashboard. To test the pipeline, send an invoice email to the address shown in your account, or use the example invoices in the `example_inputs` folder.


# Team Contributions

We collaborated across all areas of the project, with work divided pragmatically based on what needed doing at each stage. All members contributed to feature development, designing the data model, integrating external services, debugging, and testing the end-to-end pipeline with real invoices.

A significant portion of the work is not visible in the codebase itself. This includes: configuring Supabase (storage buckets, row-level security policies, database schema design), setting up the custom domain via GoDaddy, deploying and managing the backend and frontend on Railway, integrating AgentMail for per-user email inbox provisioning, and building the LangChain agent pipeline for invoice parsing and structured data extraction.

# Dataset Used
The system processes invoice documents uploaded by users. No proprietary or third-party training datasets were used — our system uses open source models, so no training data was required.
Example invoices used to test the system are included in the `example_inputs` folder.
